## 완전탐색 (Exhaustive Search)

정답이 될 가능성이 있는 모든 후보(candidates)를 체계적(반복문, 재귀, 비트마스크 등)으로 확인하는 것

## 상태 공간 트리 (State Space Tree)

상태 공간 트리는 모든 경우의 수(모든 가능한 상태)를 트리 형태로 나타낸 것으로, 이를 DFS로 탐색하면 가능한 모든 경우를 확인할 수 있다.

1. 상태 (State / Node)
   문제 해결 과정 중 어느 한순간의 상황(상태)을 의미. **특정 순간의 문제 상황을 정의**하는 모든 정보의 집합. 재귀 함수에서는 매개변수(parameter)들이 이 상태를 정의한다.
2. 상태 전이 (Edge)
   하나의 상태에서 다른 상태로 변화하는 것을 의미한다. 상태 전이는 주로 재귀 호출을 통해 일어난다.
   **현재 상태를 나타내는 매개변수 값을 변경하여 다음 함수를 호출하는 행위**자체가 바로 '상태 전이'를 구현하는 것이다.
   > 상태를 변화시킨 후 재귀를 호출 하지만, 재귀호출이 끝나면 상태는 원상복귀 시켜줘야 된다.
3. 초기 상태 (Root Node / Initial State)
   문제 해결이 시작되는 최초의 상태를 말한다. 모든 탐색은 이 초기 상태에서부터 시작되며, 문제의 가장 처음 상황에 해당한다.
4. 최종 상태 (Leaf Node)
   탐색 알고리즘의 **목표**가 되는 '상황' 이나 '조건'을 의미한다. 최종 상태는 재귀 함수에서 **Base Case(종료 조건)**에 해당하는 경우가 대부분이다.

## 순열 (Permutation)

순열은 주어진 n개의 원소 중에서 r개를 선택하여 **순서를 고려**하여 배열하는 방법의 수

## 조합 (combination)

조합은 주어진 n개의 원소 중에서 r개를 선택하여 **순서를 고려하지 않고** 구성하는 방법의 수 입니다.

## 부분 집합

부분집합은 주어진 집합에서 원소를 선택하여 **만들 수 있는 모든 경우의 집합**을 의미한다.
여기에는 공집합과 자기 자신도 포함된다.

부분집합은 결국 "n개의 원소 중에서 0개를 뽑는 조합, 1개를 뽑는 조합, 2개를 뽑는 조합, ... n개를 뽑는 조합을 모두 합친것"과 같다.
